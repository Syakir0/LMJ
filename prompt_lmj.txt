MESIN 1 — Windows (Laptop/Host OS):
  ▸ Fungsi: Development + Serving Laravel 11 Dashboard
  ▸ Tools: Laragon (PHP 8.3, Composer, MySQL lokal untuk dev) + 
    Node.js 20 LTS + Git + VS Code
  ▸ Laravel dijalankan via: php artisan serve (dev) atau 
    Laragon virtual host (produksi lokal)
  ▸ IP Windows pada adapter Host-Only: 192.168.10.10 
    (sesuaikan dengan IP yang diberikan VirtualBox/VMware)

MESIN 2 — Debian 12 "Bookworm" (VM atau fisik terpisah):
  ▸ Fungsi: MURNI backend — FreeRADIUS + MariaDB + Redis
  ▸ TIDAK ada Nginx/Apache untuk Laravel di Debian
  ▸ Interface enp0s3: NAT ke internet (IP DHCP)
  ▸ Interface enp0s8: Host-Only, IP statis 192.168.10.2
  ▸ Semua service bind ke 192.168.10.2, BUKAN 0.0.0.0

MESIN 3 — MikroTik hAP RB951Ui-2nD:
  ▸ IP management: 192.168.10.1 (terhubung ke enp0s8 Debian)
  ▸ bridge-pppoe: gabungan Ether 2-5 untuk PPPoE server
  ▸ Ether1: uplink internet
  ▸ RouterOS 7.x

MESIN 4 — Router Tenda (CPE Pelanggan):
  ▸ Mode: PPPoE Client dial ke MikroTik
  ▸ Tidak memiliki API — monitoring via ICMP + radacct

KONEKSI ANTAR MESIN:
  Windows → Debian (192.168.10.2):
    - TCP 3306: Laravel akses MariaDB
    - TCP 6379: Laravel akses Redis (queue/cache)
  Windows → MikroTik (192.168.10.1):
    - TCP 8728: Laravel akses RouterOS API
  Debian → MikroTik:
    - UDP 1812/1813: FreeRADIUS auth/accounting
  Laravel (Windows) → Telegram:
    - HTTPS ke api.telegram.org

1.1 Setup Windows (Laravel Host)
  Berikan panduan instalasi step-by-step untuk Windows:

  A. Laragon (rekomendasi utama vs XAMPP — jelaskan keunggulan Laragon 
     untuk development Laravel di Windows):
     ▸ Download Laragon Full (sudah include PHP 8.3, MySQL, Nginx, Redis).
     ▸ Konfigurasi PHP extensions yang dibutuhkan: php_pdo_mysql, 
       php_mbstring, php_xml, php_curl, php_redis, php_fileinfo, 
       php_openssl — cara enable via php.ini di Laragon.
     ▸ Cara tambahkan PHP ke PATH Windows (agar bisa `php` di CMD/PowerShell).
     ▸ Konfigurasi Composer setelah Laragon.

  B. Node.js & npm:
     ▸ Install Node.js 20 LTS dari nodejs.org.
     ▸ Verifikasi: node -v && npm -v

  C. Git for Windows:
     ▸ Install + konfigurasi user.name dan user.email.
     ▸ Generate SSH key untuk GitHub (opsional).

  D. Inisialisasi Project Laravel:
     ▸ composer create-project laravel/laravel lmj-v2 "^11.0"
     ▸ cd lmj-v2
     ▸ Install semua dependencies sekaligus:
       composer require evilnet/mikrotik-api 
                        irazasyed/telegram-bot-sdk 
                        darkaonline/l5-swagger 
                        spatie/laravel-permission 
                        spatie/laravel-activitylog 
                        league/csv 
                        laravel/sanctum 
                        laravel/horizon
     ▸ npm install && npm run build

1.2 Konfigurasi .env Windows (KRITIS — Dual OS)
  Berikan file .env LENGKAP yang sudah disesuaikan untuk koneksi remote:

  APP_NAME="LMJ-V2 ISP Dashboard"
  APP_ENV=local
  APP_DEBUG=true
  APP_URL=http://localhost:8000

  # Database: MariaDB di Debian — BUKAN localhost
  DB_CONNECTION=mysql
  DB_HOST=192.168.10.2        ← IP enp0s8 Debian
  DB_PORT=3306
  DB_DATABASE=lmj_dashboard
  DB_USERNAME=lmj_user
  DB_PASSWORD=<DB_PASSWORD>

  # Database RADIUS (koneksi terpisah ke DB radius)
  RADIUS_DB_HOST=192.168.10.2
  RADIUS_DB_PORT=3306
  RADIUS_DB_DATABASE=radius
  RADIUS_DB_USERNAME=radius_ro ← read-write untuk CRUD
  RADIUS_DB_PASSWORD=<RADIUS_DB_PASSWORD>

  # Redis: juga di Debian
  REDIS_HOST=192.168.10.2
  REDIS_PORT=6379
  REDIS_PASSWORD=null

  # Queue: gunakan Redis (bukan sync — agar job berjalan di background)
  QUEUE_CONNECTION=redis
  CACHE_STORE=redis
  SESSION_DRIVER=file          ← tetap file untuk simplicity

  # MikroTik API
  MIKROTIK_HOST=192.168.10.1
  MIKROTIK_USERNAME=admin
  MIKROTIK_PASSWORD=<MIKROTIK_PASSWORD>
  MIKROTIK_PORT=8728
  MIKROTIK_TIMEOUT=3

  # Telegram
  TELEGRAM_BOT_TOKEN=<BOT_TOKEN>
  TELEGRAM_CHAT_ID=<CHAT_ID>
  TELEGRAM_ADMIN_CHAT_ID=<ADMIN_CHAT_ID>
  TELEGRAM_NOC_CHAT_ID=<NOC_CHAT_ID>

  PENTING: Berikan penjelasan mengapa ada DUA koneksi database 
  terpisah (lmj_dashboard dan radius) dan bagaimana cara 
  konfigurasi multi-database di config/database.php Laravel 11.

1.3 Konfigurasi config/database.php — Multiple Connections
  Berikan konfigurasi lengkap untuk dua koneksi MySQL:
  ▸ 'mysql' (default): untuk tabel lmj_dashboard 
    (customers, packages, devices, alerts, dll).
  ▸ 'radius': untuk tabel FreeRADIUS 
    (radcheck, radreply, radacct, dll).
  ▸ Cara menggunakan koneksi spesifik di Model:
    protected $connection = 'radius';
  ▸ Cara menggunakan di Query Builder:
    DB::connection('radius')->table('radacct')->...

1.4 Setup Debian 12 (Backend Only)
  Berikan script bash idempoten untuk:
  ▸ Install semua paket: freeradius freeradius-mysql 
    mariadb-server redis-server ufw fping snmpd
  ▸ Konfigurasi enp0s8 dengan IP statis 192.168.10.2 
    via /etc/network/interfaces.
  ▸ UFW rules — HANYA izinkan koneksi yang dibutuhkan:
    - Port 22 (SSH): dari IP Windows saja
    - Port 1812/1813 UDP: dari 192.168.10.1 (MikroTik) saja
    - Port 3306 TCP: dari 192.168.10.10 (Windows) saja
    - Port 6379 TCP: dari 192.168.10.10 (Windows) saja
    - ICMP: dari 192.168.10.0/24 saja
    - Default: deny incoming
  ▸ Konfigurasi Redis untuk menerima koneksi dari Windows:
    Edit /etc/redis/redis.conf:
    bind 127.0.0.1 192.168.10.2
    protected-mode yes
    requirepass <REDIS_PASSWORD>   ← tambahkan password jika diperlukan

1.5 Konfigurasi MariaDB — Remote Access dari Windows
  ▸ /etc/mysql/mariadb.conf.d/50-server.cnf:
    bind-address = 192.168.10.2   ← bukan 0.0.0.0
  ▸ Buat dua database:
    CREATE DATABASE radius;
    CREATE DATABASE lmj_dashboard;
  ▸ Buat user dengan akses spesifik dari IP Windows:
    CREATE USER 'radius_rw'@'192.168.10.10' IDENTIFIED BY '<PASSWORD>';
    GRANT SELECT,INSERT,UPDATE,DELETE ON radius.* TO 'radius_rw'@'192.168.10.10';
    CREATE USER 'lmj_user'@'192.168.10.10' IDENTIFIED BY '<PASSWORD>';
    GRANT ALL PRIVILEGES ON lmj_dashboard.* TO 'lmj_user'@'192.168.10.10';
  ▸ Import schema FreeRADIUS ke database radius.
  ▸ Cara test koneksi dari Windows:
    mysql -h 192.168.10.2 -u lmj_user -p lmj_dashboard
    (install MySQL CLI di Windows atau gunakan TablePlus/DBeaver)

1.6 Troubleshooting Koneksi Windows ↔ Debian (PALING SERING BERMASALAH)
  Berikan checklist verifikasi urut dari yang paling dasar:
  Step 1: ping 192.168.10.2 dari Windows CMD → harus reply
  Step 2: Test-NetConnection 192.168.10.2 -Port 3306 (PowerShell)
          → harus TcpTestSucceeded: True
  Step 3: Test-NetConnection 192.168.10.2 -Port 6379
  Step 4: php artisan db:show (dari direktori lmj-v2 di Windows)
  Step 5: php artisan tinker → DB::connection('radius')->select('SELECT 1')
  
  Untuk setiap step, berikan:
  ▸ Output jika berhasil.
  ▸ Output jika gagal + penyebab paling umum + cara perbaiki.
  
  Penyebab paling umum yang harus dibahas:
  ▸ VM VirtualBox: adapter Host-Only belum dikonfigurasi dengan benar.
  ▸ UFW Debian memblokir port (cara debug: ufw status verbose).
  ▸ MariaDB masih bind ke 127.0.0.1 (cara cek: ss -tlnp | grep 3306).
  ▸ User MariaDB tidak punya akses dari IP Windows 
    (cara cek: SELECT user,host FROM mysql.user;).
  ▸ Windows Firewall memblokir outgoing ke port 3306/6379.

  Tambahan penting untuk Windows environment:
  ▸ Queue worker di Windows:
    php artisan queue:work redis --queue=monitoring,notifications,default
    Cara jalankan di background Windows tanpa tutup terminal:
    Gunakan Windows Task Scheduler atau NSSM (Non-Sucking Service Manager) 
    untuk jadikan queue worker sebagai Windows Service.
    Berikan panduan setup NSSM agar queue:work jalan otomatis saat startup.
  
  ▸ Laravel Scheduler di Windows:
    Tambahkan ke Windows Task Scheduler (bukan crontab):
    - Program: php
    - Arguments: C:\laragon\www\lmj-v2\artisan schedule:run
    - Interval: setiap 1 menit
    Berikan langkah GUI Task Scheduler untuk ini.
  
  ▸ Artisan commands penting yang dijalankan dari Windows CMD/PowerShell:
    php artisan device:check-all
    php artisan telegram:test
    php artisan db:show --database=radius
    php artisan horizon (untuk monitoring queue — buka di browser)

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  [Sama seperti sebelumnya]

  Perbedaan penting karena Laravel di Windows:
  ▸ MikroTikService::connect() menghubungkan Windows langsung ke 
    MikroTik 192.168.10.1:8728 via TCP.
  ▸ Tidak ada "relay" melalui Debian — Windows koneksi langsung.
  ▸ Pastikan Windows Firewall tidak blokir outgoing ke 192.168.10.1:8728.

  Untuk ICMP ping monitoring Tenda dari Windows:
  ▸ PHP exec('ping -n 3 <IP>') di Windows berbeda sintaksnya 
    dengan Linux (-c vs -n). 
    Berikan kode PHP yang cross-platform:
    $os = PHP_OS_FAMILY;
    $cmd = $os === 'Windows' 
      ? "ping -n 3 {$ip}" 
      : "fping -c 3 {$ip}";
  ▸ Alternatif: gunakan package react/dns atau library ping 
    yang cross-platform.

  ▸ Jelaskan justifikasi arsitektur dual-OS dalam laporan:
    "Pemisahan antara server backend (Debian) dan server 
     aplikasi web (Windows) mencerminkan prinsip separation

     Debian berfungsi sebagai dedicated authentication server

     dari internet publik, meningkatkan keamanan sistem
