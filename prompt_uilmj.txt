2.3 Reusable Blade Components
Berikan kode untuk setiap component di components/:

  A. <x-stat-card>
     Props: title, value, subtitle, icon, color, trend (up/down/neutral)
     Tampilan: icon bulat di kiri, value besar di tengah,
     trend arrow + persentase perubahan di bawah.
     Contoh penggunaan:
     <x-stat-card
       title="Pelanggan Online"
       :value="$onlineCount"
       icon="wifi"
       color="success"
       trend="up"
       trend-value="+3 dari kemarin"
     />

  B. <x-device-status>
     Props: status (online/offline/degraded/unknown), show-label
     Tampilan: dot berwarna + animasi pulse jika online +
     label teks opsional.

  C. <x-alert-badge>
     Props: severity (info/warning/critical), count (opsional)
     Tampilan: pill badge dengan warna sesuai severity.

  D. <x-modal>
     Props: id, title, size (sm/md/lg/xl)
     Alpine.js controlled: x-show + @keydown.escape.window
     Backdrop click untuk close.
     Slot: default (body), footer (tombol aksi).

9.2 Toast Notifications
  Berikan komponen toast global (Alpine.js + Tailwind):
  ▸ Posisi: bottom-right, stack ke atas
  ▸ Tipe: success (hijau), error (merah), warning (kuning), info (biru)
  ▸ Auto-dismiss: 4 detik + manual close button
  ▸ Trigger dari mana saja:

      detail: { type: 'success', message: 'User berhasil diputus' }
    }))

9.3 Confirm Dialogs
  ▸ Semua aksi destruktif (Hapus, Disconnect, Suspend) WAJIB
    tampilkan confirm modal bukan browser confirm().
  ▸ Confirm modal: judul merah, ikon warning, deskripsi aksi,
    [Batal] [Ya, Lanjutkan] (tombol merah)

9.4 Empty States
  ▸ Jika tabel/grid kosong: tampilkan ilustrasi SVG sederhana
    + pesan kontekstual + tombol aksi (misal "Tambah Pelanggan Pertama")
  ▸ Jangan tampilkan tabel kosong tanpa penjelasan

9.5 Error Handling
  ▸ Jika API request gagal (network error, 500):
    tampilkan error toast + retry button di section yang gagal
  ▸ Jangan biarkan halaman blank atau error tak tertangani

9.6 Responsive Breakpoints
  ▸ Mobile (< 768px): sidebar jadi drawer, tabel horizontal scroll,
    charts ukuran lebih kecil, grid → 1 kolom
  ▸ Tablet (768-1024px): sidebar collapse, grid → 2 kolom
  ▸ Desktop (> 1024px): full layout seperti desain di atas
